Welcome to Simon JS.

Simon JS is a Single Page Application that helps you monitor the status of your development infrastructure.

It is meant as a tool to help developers gain a quick overview and easy access to build jobs on applications like Jenkins, software quality projects on tools like Sonarqube and services on infrastructure monitoring tools like Nagios.

How does Simon JS work?

Simon JS is a Single Page Application using JavaScript to get remote data and display it on the page. It's based on AMD/Require JS uses Knockout for view rendering, jQuery, Moment.js and Twitter Bootstrap for layout and frontent effects. While Simon JS is written in Typescript, the Simon JS library is distributed in JavaScript.

Data from backend servers is loaded as JSON, applied to JavaScript models which are in turn displayed on the page using Knockout. The data is reloaded periodically (currently every 60-120 seconds). Thanks to Knockout's two way data binding, the page is seamlessly updated with new data.

  • open page from http://www.simonjs.org/release/1.0.0/index.html

  • supply your own configuration using URL parameter "config": http://www.simonjs.org/release/1.0.0/index.html?config=http://your-server/config.js

  • Simon JS is configured using a JSON object stored in a global variable "configJson": configJson={}

  • The configuration object must be compatible to the Typescript interface https://github.com/afranken/simon/blob/master/src/main/ts/jsonInterfaces/Config.d.ts
  • When you open http://www.simonjs.org/release/1.0.0/index.html, a sample configuration is used that can be found here: http://www.simonjs.org/release/1.0.0/config.js
